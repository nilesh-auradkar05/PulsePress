PulsePress — Component / Bounded Contexts v1.2

Two write-shapes, one async backbone, no decorative event-driven nonsense.

Publishing context

Content-shaped. Normal CRUD is synchronous and quiet.

  • publications
  • posts
  • post_versions on publish
  • newsletter_sends via worker

Only event door: publishPost → post.published

Commerce context

Money-shaped. Every commerce mutation requires idempotency and outbox discipline.

  • subscriptions
  • subscription_plans
  • gift_transactions
  • ledger_transactions + ledger_entries via worker

Agent rule

If a task tries to route ordinary post edits through the outbox, reject it. If a commerce mutation lacks idempotency/outbox, reject it harder.
