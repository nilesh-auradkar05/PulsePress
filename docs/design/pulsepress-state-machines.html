PulsePress — State Machines v1.2

Post, subscription, and gift lifecycles after idempotency/tier-change cleanup.

Post lifecycle

draft → published → archived

  • CRUD self-loops emit no event.
  • publishPost emits post.published.
  • No republish in Phase 1.

Subscription lifecycle

active → canceled → expired

  • Subscribe, tier change, cancel require idempotency.
  • Tier change immediate, no ledger in P1.
  • Expired is lazy-derived.

Gift lifecycle

pending → processed | failed

  • One-shot money event.
  • Same ledger model as paid subscription.
  • Admin retry handles failures.
Event list uses ledger.transaction.recorded, the former row-level ledger event name.
